# Linksee MVP P0 任务与后续跟进任务

## 1. 目标

本文用于明确两个月内必须完成的 P0 范围，并把容易扩大项目体量的能力放入后续任务池。

当前 MVP 目标：

> 完成“老师发布课程项目阶段要求 -> 学生小组协作 -> 小组提交成果 -> 老师/助教查看过程并批改反馈”的闭环。

第一版以课程项目为主，不展开非课程组队、大创竞赛、复杂教务系统、AI 自动评审等能力。

## 2. P0 范围分层

为保证两个月内可交付，P0 不再理解为“所有重要功能都必须完整实现”，而是拆成两层：

- P0-A：必须完成的演示闭环。缺少任一项，答辩链路不完整。
- P0-B：闭环稳定后补齐的增强项。可以提高说服力，但不得阻塞 P0-A。

统一术语：

| 术语 | 含义 |
| --- | --- |
| Course | 课程空间 |
| Assignment | 老师发布的课程项目 |
| Stage | Assignment 下的阶段要求，承担老师层任务职责 |
| Group | 学生项目小组 |
| MiniTask | 小组内部拆分的执行任务 |
| Submission | 小组针对 Stage 的成果提交 |
| Review | 老师或助教的反馈 |
| Grade | 阶段分数或最终成绩 |

P0 中不单独实现老师层 `Task` 实体，避免与 `Stage` 重复。老师发布的“阶段要求”统一建模为 `Stage`；学生内部任务统一建模为 `MiniTask`。

## 3. P0-A 必做任务

### 3.1 账号与角色

目标：用户可以登录，并根据角色进入不同工作台。

任务：

- 实现账号登录。
- 支持老师、学生、助教三类角色。
- 注册时确定用户角色。
- 登录后根据角色进入不同主界面。
- 退出登录需要确认。

验收：

- 老师登录后进入老师工作台。
- 学生登录后进入学生小组空间。
- 助教登录后进入助教检查台。
- 重新打开系统时回到登录页。

### 3.2 课程与项目

目标：老师可以创建课程项目，学生围绕课程项目协作。

任务：

- 老师创建课程。
- 老师创建课程项目。
- 老师设置项目名称、说明、起止时间。
- 老师发布项目给课程内学生。

验收：

- 老师能看到自己创建的课程项目。
- 学生能看到自己参与课程下的项目。

### 3.3 学生名单与小组

目标：课程项目下可以形成小组。

任务：

- 老师导入或手动添加学生名单。
- 老师创建小组。
- 学生加入小组。
- 学生在同一项目中只能属于一个小组。
- 老师可以调整学生所在小组。

验收：

- 老师能看到课程内学生和小组列表。
- 学生能看到自己所在小组。
- 同一学生不能重复加入同一项目下多个小组。

### 3.4 小组协作空间

目标：学生不是只提交结果，而是在小组空间里协作推进项目。

任务：

- 小组 MiniTask 看板。
- 小组讨论。
- 文件/链接记录。
- 仓库链接记录。
- 小组动态记录。

验收：

- 学生能创建和推进 MiniTask。
- 学生能在小组内讨论。
- 学生能记录文件、链接和仓库地址。
- 老师/助教能查看小组协作过程。

### 3.5 阶段要求与 MiniTask

目标：区分老师布置的阶段要求和学生内部拆分的执行任务。

任务：

- 老师在 Assignment 下创建 Stage。
- Stage 包含目标描述、截止时间、提交要求、验收标准。
- 小组成员基于 Stage 创建 MiniTask。
- MiniTask 包含负责人、状态、优先级、计划时间。
- MiniTask 支持待办、进行中、已完成状态。

验收：

- 老师能发布课程项目阶段要求。
- 学生能在小组内拆分 MiniTask。
- 老师能查看每个小组的 MiniTask 推进情况。

### 3.6 成果提交

目标：小组可以按 Stage 提交成果。

任务：

- 小组提交文件材料。
- 小组提交外部链接。
- 小组提交仓库地址。
- 小组填写提交说明。
- 小组填写贡献说明。
- 提交状态支持：未提交、已提交、需修改、已重交、已通过。

验收：

- 学生能完成一次阶段提交。
- 老师能看到提交内容。
- 提交状态能被老师更新。

### 3.7 批改与反馈

目标：老师和助教可以查看提交材料并给出反馈。

任务：

- 老师查看小组提交内容。
- 助教查看小组提交内容。
- 老师/助教填写批改意见。
- 老师/助教登记阶段分数。
- 老师确认最终成绩。
- 学生查看反馈和状态。

验收：

- 老师能给提交打分并写反馈。
- 助教能辅助批改和登记。
- 学生能看到反馈。
- 最终成绩发布权保留给老师。

### 3.8 老师追踪看板

目标：老师能快速掌握课程项目推进状态。

任务：

- 展示所有小组进度。
- 展示未提交小组。
- 展示待批改提交。
- 展示需修改提交。
- 展示协作不活跃小组。
- 展示风险提醒。

验收：

- 老师打开看板后能知道哪些组需要关注。
- 老师能从看板进入小组详情。

## 4. P0-B 闭环稳定后补齐

### 4.1 助教检查台

目标：助教可以承担材料检查和批改辅助工作。

任务：

- 展示待检查提交。
- 展示材料缺失提交。
- 展示需复核提交。
- 展示助教已处理记录。

验收：

- 助教能查看自己需要处理的提交。
- 老师能看到助教处理结果。

### 4.2 基础留痕

目标：关键教学评价动作有记录可查。

任务：

- 记录提交时间。
- 记录提交人。
- 记录批改人。
- 记录反馈时间。
- 记录成绩修改前后值。
- 记录助教操作。

验收：

- 老师能查看本课程关键操作记录。
- 成绩修改有可追溯记录。

### 4.3 简化 Rubric 与成绩确认

目标：让评分更有依据，但不把复杂评分系统放进 P0-A。

任务：

- 老师配置 3 到 5 个评分项。
- 老师给每个 Stage 登记阶段分数。
- 助教可以填写建议分数。
- 老师确认最终成绩。

验收：

- 老师能按评分项给提交打分。
- 学生能查看反馈和阶段分数。
- 最终成绩发布权保留给老师。

## 5. 状态流转约定

Submission 状态统一为：

```text
not_submitted -> submitted -> needs_changes -> resubmitted -> approved
```

说明：

- `not_submitted`：到期前或尚未提交。
- `submitted`：学生首次提交，等待老师或助教查看。
- `needs_changes`：老师或助教要求修改。
- `resubmitted`：学生根据反馈重新提交。
- `approved`：老师确认通过。

P0 暂不使用 `rejected` 作为常规状态。若确实需要表达不通过，可在 Review 中记录结论，避免状态流转过复杂。

## 6. P0 不做事项

以下内容不进入两个月 MVP：

- 教务系统深度对接。
- 复杂教务处后台。
- 自动排课或统一身份认证。
- 非课程项目组队。
- 大创、竞赛、社团项目空间。
- GitHub commit 自动分析。
- GitHub Actions / GitLab CI 自动测试结果同步。
- AI 自动评分。
- RAG 文件问答。
- 复杂组号自动重排。
- 复杂审批流。
- 多学校多租户治理。

教务处角色在 P0 中可用系统管理员账号或种子数据模拟，不实现完整教务处后台。

## 7. P1 后续跟进任务

P0 跑通后，优先跟进 P1。

### 7.1 完整组队面板

- 学生创建小组。
- 学生申请加入小组。
- 组长审批加入申请。
- 组长邀请成员。
- 成员退出小组。
- 组长转让。
- 显示组号占用表。

### 7.2 组队截止与组号整理

- 设置组队开始时间和截止时间。
- 截止后自动清理空组号。
- 截止后自动连续重排展示组号。
- 老师确认最终组号。

### 7.3 课程群

- 课程公共通知。
- 课程公共讨论。
- 老师面向全班发布提醒。
- 与小组群区分权限和用途。

### 7.4 教务处基础视角

- 教务处查看课程列表。
- 教务处查看课程项目推进。
- 教务处查看批改覆盖率。
- 教务处查看成绩变更日志。

### 7.5 更完整的 Rubric

- 老师配置评分项。
- 老师配置评分权重。
- 支持阶段评分和最终评分。
- 支持评分说明模板。

### 7.6 成绩导出

- 导出小组成绩。
- 导出个人成绩。
- 导出批改记录。
- 导出过程性评价摘要。

## 8. P2 后续扩展任务

P1 稳定后，再考虑 P2。

### 8.1 非课程学生自发项目

定位：

> 支持学生在课程之外创建自发项目，例如竞赛、大创、社团、科研训练和兴趣项目。

任务：

- 支持项目类型：课程项目 / 学生自发项目。
- 学生创建非课程项目。
- 学生邀请成员加入。
- 支持公开招募或邀请制。
- 复用小组任务、讨论、文件、仓库链接能力。
- 支持项目总结和作品沉淀。

注意：

- 非课程项目不进入 P0。
- 非课程项目不默认需要老师评分。
- 后续可支持导师点评或项目认证。

### 8.2 GitHub 集成

- 绑定 GitHub 仓库。
- 同步 commit 记录。
- 展示成员提交概况。
- 展示 GitHub Actions 结果。
- 将自动代码贡献统计作为过程证据之一。

### 8.3 自动测试

- 老师配置测试说明。
- 学生提交软件包或仓库。
- 系统记录测试结果。
- 测试结果进入评分参考。

### 8.4 AI 助教

- 自动生成小组周报。
- 自动总结提交材料。
- 自动生成教师评语草稿。
- 自动识别延期和协作不活跃风险。

### 8.5 RAG 文件问答

- 文档切片。
- 向量索引。
- 课程资料问答。
- 小组材料问答。
- 回答引用来源。

### 8.6 学生成长档案

- 汇总学生参与过的课程项目。
- 汇总非课程项目。
- 汇总贡献记录。
- 生成项目作品集。
- 支持导出简历素材。

## 9. 推荐开发顺序

1. 账号与角色。
2. 课程与项目。
3. 学生名单与小组。
4. Stage 与 MiniTask。
5. 小组协作空间。
6. 成果提交。
7. 批改与反馈。
8. 老师追踪看板。
9. P0-B 助教检查台。
10. P0-B 基础留痕与简化 Rubric。

## 10. 当前界面原型对应关系

当前前端 demo 已覆盖：

- 登录与注册入口。
- 老师、学生、助教三类角色主界面。
- 老师追踪看板原型。
- 学生小组任务和讨论原型。
- 助教检查台原型。

下一步建议优先把原型中的模拟数据替换为真实数据结构，而不是继续增加新页面。
