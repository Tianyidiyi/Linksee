# Linksee MVP P0 任务与后续跟进任务

## 1. 目标

本文用于明确两个月内必须完成的 P0 范围，并把容易扩大项目体量的能力放入后续任务池。

当前 MVP 目标：

> 完成“老师发布课程项目任务 -> 学生小组协作 -> 小组提交成果 -> 老师/助教查看过程并批改反馈”的闭环。

第一版以课程项目为主，不展开非课程组队、大创竞赛、复杂教务系统、AI 自动评审等能力。

## 2. P0 必做任务

### 2.1 账号与角色

目标：用户可以登录，并根据角色进入不同工作台。

任务：

- 实现账号登录。
- 支持老师、学生、助教三类角色。
- 注册时确定用户角色。
- 登录后根据角色进入不同主界面。
- 退出登录需要确认。

验收：

- 老师登录后进入老师工作台。
- 学生登录后进入学生小组空间。
- 助教登录后进入助教检查台。
- 重新打开系统时回到登录页。

### 2.2 课程与项目

目标：老师可以创建课程项目，学生围绕课程项目协作。

任务：

- 老师创建课程。
- 老师创建课程项目。
- 老师设置项目名称、说明、起止时间。
- 老师发布项目给课程内学生。

验收：

- 老师能看到自己创建的课程项目。
- 学生能看到自己参与课程下的项目。

### 2.3 学生名单与小组

目标：课程项目下可以形成小组。

任务：

- 老师导入或手动添加学生名单。
- 老师创建小组。
- 学生加入小组。
- 学生在同一项目中只能属于一个小组。
- 老师可以调整学生所在小组。

验收：

- 老师能看到课程内学生和小组列表。
- 学生能看到自己所在小组。
- 同一学生不能重复加入同一项目下多个小组。

### 2.4 小组协作空间

目标：学生不是只提交结果，而是在小组空间里协作推进项目。

任务：

- 小组任务看板。
- 小组讨论。
- 文件/链接记录。
- 仓库链接记录。
- 小组动态记录。

验收：

- 学生能创建和推进小组任务。
- 学生能在小组内讨论。
- 学生能记录文件、链接和仓库地址。
- 老师/助教能查看小组协作过程。

### 2.5 Task-MiniTask 两层任务机制

目标：区分老师布置的课程目标任务和学生内部拆分的执行任务。

任务：

- 老师创建 Task。
- Task 包含目标描述、截止时间、提交要求、验收标准。
- 小组成员基于 Task 创建 MiniTask。
- MiniTask 包含负责人、状态、优先级、计划时间。
- MiniTask 支持待办、进行中、已完成状态。

验收：

- 老师能发布课程级 Task。
- 学生能在小组内拆分 MiniTask。
- 老师能查看每个小组的 MiniTask 推进情况。

### 2.6 成果提交

目标：小组可以按任务提交成果。

任务：

- 小组提交文件材料。
- 小组提交外部链接。
- 小组提交仓库地址。
- 小组填写提交说明。
- 小组填写贡献说明。
- 提交状态支持：未提交、已提交、需修改、已通过。

验收：

- 学生能完成一次阶段提交。
- 老师能看到提交内容。
- 提交状态能被老师更新。

### 2.7 批改与反馈

目标：老师和助教可以查看提交材料并给出反馈。

任务：

- 老师查看小组提交内容。
- 助教查看小组提交内容。
- 老师/助教填写批改意见。
- 老师/助教登记阶段分数。
- 老师确认最终成绩。
- 学生查看反馈和状态。

验收：

- 老师能给提交打分并写反馈。
- 助教能辅助批改和登记。
- 学生能看到反馈。
- 最终成绩发布权保留给老师。

### 2.8 老师追踪看板

目标：老师能快速掌握课程项目推进状态。

任务：

- 展示所有小组进度。
- 展示未提交小组。
- 展示待批改提交。
- 展示需修改提交。
- 展示协作不活跃小组。
- 展示风险提醒。

验收：

- 老师打开看板后能知道哪些组需要关注。
- 老师能从看板进入小组详情。

### 2.9 助教检查台

目标：助教可以承担材料检查和批改辅助工作。

任务：

- 展示待检查提交。
- 展示材料缺失提交。
- 展示需复核提交。
- 展示助教已处理记录。

验收：

- 助教能查看自己需要处理的提交。
- 老师能看到助教处理结果。

### 2.10 基础留痕

目标：关键教学评价动作有记录可查。

任务：

- 记录提交时间。
- 记录提交人。
- 记录批改人。
- 记录反馈时间。
- 记录成绩修改前后值。
- 记录助教操作。

验收：

- 老师能查看本课程关键操作记录。
- 成绩修改有可追溯记录。

## 3. P0 不做事项

以下内容不进入两个月 MVP：

- 教务系统深度对接。
- 复杂教务处后台。
- 自动排课或统一身份认证。
- 非课程项目组队。
- 大创、竞赛、社团项目空间。
- GitHub commit 自动分析。
- GitHub Actions / GitLab CI 自动测试结果同步。
- AI 自动评分。
- RAG 文件问答。
- 复杂组号自动重排。
- 复杂审批流。
- 多学校多租户治理。

## 4. P1 后续跟进任务

P0 跑通后，优先跟进 P1。

### 4.1 完整组队面板

- 学生创建小组。
- 学生申请加入小组。
- 组长审批加入申请。
- 组长邀请成员。
- 成员退出小组。
- 组长转让。
- 显示组号占用表。

### 4.2 组队截止与组号整理

- 设置组队开始时间和截止时间。
- 截止后自动清理空组号。
- 截止后自动连续重排展示组号。
- 老师确认最终组号。

### 4.3 课程群

- 课程公共通知。
- 课程公共讨论。
- 老师面向全班发布提醒。
- 与小组群区分权限和用途。

### 4.4 教务处基础视角

- 教务处查看课程列表。
- 教务处查看课程项目推进。
- 教务处查看批改覆盖率。
- 教务处查看成绩变更日志。

### 4.5 更完整的 Rubric

- 老师配置评分项。
- 老师配置评分权重。
- 支持阶段评分和最终评分。
- 支持评分说明模板。

### 4.6 成绩导出

- 导出小组成绩。
- 导出个人成绩。
- 导出批改记录。
- 导出过程性评价摘要。

## 5. P2 后续扩展任务

P1 稳定后，再考虑 P2。

### 5.1 非课程学生自发项目

定位：

> 支持学生在课程之外创建自发项目，例如竞赛、大创、社团、科研训练和兴趣项目。

任务：

- 支持项目类型：课程项目 / 学生自发项目。
- 学生创建非课程项目。
- 学生邀请成员加入。
- 支持公开招募或邀请制。
- 复用小组任务、讨论、文件、仓库链接能力。
- 支持项目总结和作品沉淀。

注意：

- 非课程项目不进入 P0。
- 非课程项目不默认需要老师评分。
- 后续可支持导师点评或项目认证。

### 5.2 GitHub 集成

- 绑定 GitHub 仓库。
- 同步 commit 记录。
- 展示成员提交概况。
- 展示 GitHub Actions 结果。
- 将代码贡献作为过程证据之一。

### 5.3 自动测试

- 老师配置测试说明。
- 学生提交软件包或仓库。
- 系统记录测试结果。
- 测试结果进入评分参考。

### 5.4 AI 助教

- 自动生成小组周报。
- 自动总结提交材料。
- 自动生成教师评语草稿。
- 自动识别延期和协作不活跃风险。

### 5.5 RAG 文件问答

- 文档切片。
- 向量索引。
- 课程资料问答。
- 小组材料问答。
- 回答引用来源。

### 5.6 学生成长档案

- 汇总学生参与过的课程项目。
- 汇总非课程项目。
- 汇总贡献记录。
- 生成项目作品集。
- 支持导出简历素材。

## 6. 推荐开发顺序

1. 账号与角色。
2. 课程与项目。
3. 学生名单与小组。
4. Task-MiniTask。
5. 小组协作空间。
6. 成果提交。
7. 批改与反馈。
8. 老师追踪看板。
9. 助教检查台。
10. 基础留痕。

## 7. 当前界面原型对应关系

当前前端 demo 已覆盖：

- 登录与注册入口。
- 老师、学生、助教三类角色主界面。
- 老师追踪看板原型。
- 学生小组任务和讨论原型。
- 助教检查台原型。

下一步建议优先把原型中的模拟数据替换为真实数据结构，而不是继续增加新页面。
