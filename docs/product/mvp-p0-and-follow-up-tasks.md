# MVP P0 与后续任务清单（课程协作方向）

> 更新时间：2026-05-11  
> 目标：给当前版本一个可执行、可验收的任务视图。

## 1. P0（当前主线）

### 1.1 已完成（后端）
- 课程管理：课程 CRUD、教师绑定、助教绑定、学生导入。
- 作业管理：Assignment CRUD、说明附件上传/删除（MinIO）。
- 阶段管理：Stage CRUD、状态流转、要求附件上传/删除（MinIO）。
- 组队管理：
  - 小组列表与创建；
  - 手动增删组员；
  - 入组申请（申请/查看/同意/拒绝）；
  - 组长转让（发起/接受/拒绝，双向确认）。
- 协作消息：课程/小组会话消息接口已开放。

### 1.2 P0 未完成（建议继续）
- 组队自动重排（截止后第一轮、教师确认后第二轮）。
- 教学侧批量调组能力（合并组、批量补位）。
- 提交与评审链路的完整规则校验收口（和教学策略完全对齐）。

## 2. P1（建议下一阶段）
- MiniTask 前端实现与联调（后端 API 已落地，补前端与验收）。
- 组队异常处理机制（冲突提示、可操作入口、流程状态可视化）。
- 评分维度（Rubric）配置与评审接口进一步标准化。
- 课程 Dashboard 指标接口（小组进度、待评审、超期项）。

## 3. P2（增强与工程化）
- GitHub Classroom / 提交仓库联动能力。
- AI 辅助评审与知识问答（RAG）稳定化上线。
- 更完整的自动化测试矩阵（权限、流程、边界、并发）。

## 4. 组队专项后续任务（基于本次实现）
- 增加“学生取消申请”接口（当前无）。
- 增加“组长主动取消转让请求”接口（当前无）。
- 增加“截止后仅教学侧可调整”的专门路由与前端状态提示对齐。
- 为组队流程补齐单测/集成测（含并发审批与容量边界）。

## 5. 验收建议（简版）
- 产品验收：按“学生、组长、教师/助教”三条主路径走通。
- 技术验收：接口文档、权限校验、错误码语义三者一致。
- 交付验收：前后端对齐文档与 OpenAPI 同步更新。
