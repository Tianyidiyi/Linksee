# Frontend-Backend Alignment（v1）

前后端联调以“后端已开放并可运行接口”为准，避免按历史草案开发。

## 1. 当前可对齐的后端能力

1. Auth + Users
   - 登录、刷新、登出、改密
   - 当前用户资料读写
   - 头像上传（MinIO）
2. Courses
   - 课程列表/详情/更新
   - 教师关系管理（含 lead/co）
   - 助教绑定、解绑与查询
   - 课程成员查询、单个导入、批量导入、移除
3. Assignments + Stages
   - Assignment 顶层 CRUD 与状态变更
   - Stage CRUD 与状态变更
   - Assignment/Stage 附件上传与删除（统一 MinIO 规范）

## 2. 联调约束

1. ID 统一按字符串处理（后端 BigInt 序列化为 string）。
2. 附件上传统一 `multipart/form-data`，字段名 `files`。
3. Assignment 状态：`draft | active | archived`。
4. Stage 状态：`planned | open | closed | archived`。
5. 学生端默认不显示 `draft` assignment 与 `planned` stage。
