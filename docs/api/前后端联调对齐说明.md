## 1. 使用方式
- 历史联调项继续保留，避免丢失需求背景。
- 每个能力项标注状态：

## 3. 当前联调主线（建议）
- 先走已实现主链路：
  1. 课程、教师/助教、成员
  2. 作业与阶段
  3. 小组列表/创建/成员维护
  4. 入组申请与组长转让
  5. 课程/小组消息基础收发
- 对“未实现”能力统一前端降级处理：隐藏入口或标记“待上线”。

## 4. 关键规则（当前后端已生效）
- 学生组队操作受 `groupFormEnd` 截止限制。
- 同一学生在同一作业下只能在一个组。
- 学生自助入组受 `groupMaxSize` 限制。
- 组长转让需要目标成员确认，且同组同一时间仅一条待处理转让请求。

## 5. 联调注意项
- 所有路径 ID 按字符串传输（后端内部转 BigInt）。
- 文件接口使用 `multipart/form-data`。
- Assignment 状态：`draft | active | archived`
- Stage 状态：`planned | open | closed | archived`
