# Frontend-Backend Alignment（v1）

前后端联调以“后端已开放并可运行接口”为准，避免按历史草案开发。
接口实现与可联调范围以 [docs/status/current-delivery-status.md](docs/status/current-delivery-status.md) 为准。

## 1. 当前可对齐的后端能力

1. Auth + Users
   - 登录、刷新、登出、改密
   - 当前用户资料读写
   - 头像上传（MinIO）
2. Courses
   - 课程列表/详情/更新
   - 教师关系管理（含 lead/co）
   - 助教绑定、解绑与查询
   - 课程成员查询、单个导入、批量导入、移除
3. Assignments + Stages
   - Assignment 顶层 CRUD 与状态变更
   - Stage CRUD 与状态变更
   - Assignment/Stage 附件上传与删除（统一 MinIO 规范）
4. Collaboration + Groups
   - 课程群消息（GET/POST /api/v1/courses/:courseId/messages）
   - 课程群消息搜索/编辑/撤回（GET /courses/:courseId/messages/search，PATCH/DELETE /courses/:courseId/messages/:messageId）
   - 课程公告（POST /courses/:courseId/announcements）
   - 小组群消息（GET/POST /api/v1/groups/:groupId/messages）
   - 小组群消息搜索/编辑/撤回（GET /groups/:groupId/messages/search，PATCH/DELETE /groups/:groupId/messages/:messageId）
   - 小组公告（POST /groups/:groupId/announcements）
   - 会话列表与未读（GET /conversations，POST /conversations/:conversationId/read）
   - 群聊文件签名上传/下载（POST /chat/files/presign-upload，GET /chat/files/presign-download）
   - Realtime ACK/重放（POST /realtime/acks，GET /realtime/replay）
   - 小组列表（GET /api/v1/assignments/:assignmentId/groups）
   - 小组创建（POST /api/v1/assignments/:assignmentId/groups）
   - 小组成员增删（POST/DELETE /api/v1/groups/:groupId/members）

## 2. 联调约束

1. ID 统一按字符串处理（后端 BigInt 序列化为 string）。
2. 附件上传统一 `multipart/form-data`，字段名 `files`。
3. Assignment 状态：`draft | active | archived`。
4. Stage 状态：`planned | open | closed | archived`。
5. 学生端默认不显示 `draft` assignment 与 `planned` stage。
