# 新人首次安装步骤与日常命令清单

## 1. 新人首次安装步骤

### 1.1 安装 Node.js（Windows）

1. 下载 Node LTS（建议 20.x）。
2. 安装后打开新终端验证：
- node -v
- npm -v

### 1.2 安装 Docker Desktop

1. 安装并启动 Docker Desktop。
2. 验证：
- docker --version
- docker info

### 1.3 拉起项目依赖

在仓库根目录执行：

1. 安装依赖：
- npm install

2. 复制并检查 Docker 环境变量：
- Windows PowerShell: `Copy-Item infra/docker/.env.example infra/docker/.env`
- macOS/Linux: `cp infra/docker/.env.example infra/docker/.env`
- 按需修改密码

3. 启动 MySQL/Redis/MinIO：
- npm run dev:deps:up

4. 查看容器状态：
- docker compose -f infra/docker/docker-compose.dev.yml ps

### 1.4 跑构建打包验证

- npm run verify:build

说明：

- verify:build 是项目自定义脚本，不是 npm 官方内置命令。
- 定义位置：仓库根目录 package.json 的 scripts 字段。

### 1.5 启动 Demo

- npm run demo:start

打开：
- https://localhost:3443/login.html
- https://localhost:3443/status.html

若浏览器提示证书不受信任，继续访问即可（自签名证书）。

## 2. 日常命令清单（开发期）

### 2.1 开工前

1. 启动依赖：
- npm run dev:deps:up

2. 看容器是否 healthy：
- docker compose -f infra/docker/docker-compose.dev.yml ps

### 2.2 开发中

1. 需要验证 Demo：
- npm run demo:start

2. 验证 Socket 在线状态链路：
- npm run test:socket-demo -w @clickdown/api

说明：产品名已调整为 Linksee，但当前 workspace 包名暂沿用历史名称 `@clickdown/api`。

### 2.3 提交前

1. 构建和打包：
- npm run verify:build

### 2.4 收工后

1. 关闭依赖容器：
- npm run dev:deps:down

## 3. 常见问题

### 3.1 页面跳到 file:///C:/status.html

原因：直接用文件方式打开了 html，而不是通过 demo 服务地址访问。

正确方式：
- 访问 https://localhost:3443/login.html 和 https://localhost:3443/status.html

### 3.2 docker info 报 daemon 未运行

处理：
1. 打开 Docker Desktop。
2. 等右下角显示运行中后重试 docker info。

### 3.3 命令拼写问题

正确命令：
- npm run verify:build

不是：
- vertify:build
