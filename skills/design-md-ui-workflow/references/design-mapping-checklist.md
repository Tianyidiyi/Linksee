## 执行步骤

1. 从 [docs/UI设计参考/awesome-design-md/design-md](../../../docs/UI设计参考/awesome-design-md/design-md/) 中选择 1 到 2 个风格作为基线。
2. 提取核心设计变量：色板、字体、间距、圆角、阴影、动效节奏。
3. 将变量映射到项目可复用层：`packages/ui-kit` 或 [apps/web](../../../apps/web/) 的主题文件。
4. 按下方"页面优先级"顺序推进，与后端 API 开发节奏对齐（第 1 轮先做 P0-A 页面，可用 mock 数据）。
5. 页面实现优先复用组件，不在业务页面硬编码样式。
6. 在 PR 中附上设计来源与映射说明，便于复盘和迭代。

## Linksee 页面详细设计重点

### 角色公共

| 页面 | 设计重点 |
| ---- | -------- |
| 登录页 | 账号密码清晰，登录后按 role 自动跳转对应工作台（teacher → 老师看板，student → 课程列表） |
| 课程列表 | 卡片展示课程名/课程号/学期，状态徽章（进行中/已归档），角色不同展示不同操作入口 |

### 老师 / 助教

| 页面 | 设计重点 |
| ---- | -------- |
| 老师看板 | 核心是 4 个快扫区：未提交小组 / 待批改 / 已延期 / 协作不活跃；数字卡片 + 可点击进入小组详情 |
| Assignment 管理 | 创建/编辑项目：标题、描述、文件附件、组队规则配置（人数上下限、最大组数、截止时间） |
| Stage 管理 | 阶段列表按 stage_no 排序；每阶段含：截止时间、权重、状态徽章（planned/open/closed）、提交要求文件 |
| 老师划分组 | 表格化：左侧学生名单（未分组高亮），右侧小组格子；拖拽或下拉选择把学生分配到指定组 |
| 批改台 | 左：提交物预览（文件列表 + 链接 + 贡献说明）；右：Review 输入 + 评分 + 状态操作；两栏布局 |
| 助教检查台 | 筛选栏（待检查 / 缺材料 / 需复核 / 已处理）+ 列表；操作仅批改，不发布最终成绩 |

### 学生

| 页面 | 设计重点 |
| ---- | -------- |
| Assignment 详情 | 当前项目阶段进度条（stage_no 顺序）；我的小组状态；进入小组空间的主要入口 |
| 小组空间 | 四大模块并列：MiniTask 看板 / 项目组群聊 / 文件列表 / Stage 提交入口；当前 Stage 要求醒目展示 |
| MiniTask 看板 | 三栏（待开始/进行中/已完成）+ 任务卡片（负责人头像、截止时间、优先级色标）；可拖拽状态 |
| 项目组群聊 | 消息流（时间戳 + 头像 + 气泡）+ 文件上传；历史消息分页加载；实时推送靠 socket `group:*` 房间 |
| 提交台 | 当前 Stage 要求说明 → 上传文件（多文件）→ 填写仓库链接 → 贡献说明 → 提交；状态流转明确展示 |
| 学生组队面板（P0-B） | 表格展示所有组号位：已有成员/组长/是否可加入；空位为灰色占位格；先占先得交互 |

## 通用检查清单

- [ ] 颜色与字体以变量形式管理（不硬编码十六进制值）
- [ ] 组件状态齐全：default、hover、active、disabled、loading、error
- [ ] 三类角色（teacher/student/assistant）登录后主路径入口清晰，不混用
- [ ] 所有列表页支持空态（暂无数据）展示，不留白屏
- [ ] 状态徽章（planned/open/closed/archived/submitted/approved 等）颜色语义一致
- [ ] 文件上传组件可复用（提交台/Stage 附件/群聊附件共用一套）
- [ ] 移动端可用，关键断点已验证（小组空间四模块在窄屏下折叠为 Tab 切换）
- [ ] 页面动效有节奏且不过度（状态切换 150ms，列表加载 skeleton 不闪烁）
- [ ] 无明显视觉漂移或风格混搭
- [ ] 页面没有做成营销落地页，首屏直接服务核心任务
- [ ] 老师看板的数字卡片在数据未加载时展示 skeleton，不展示 0
- [ ] socket 实时消息到达时有轻量提示（角标/气泡），不打断用户当前操作
