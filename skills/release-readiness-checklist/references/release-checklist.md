# 发版就绪检查清单

## 适用场景

- Sprint 结束发版
- 紧急修复上线

## 执行步骤

1. 确认代码状态：关键 PR 已合并，冲突已解决。
2. 确认质量门禁：lint、测试、构建通过。
3. 确认配置变更：环境变量、密钥、数据库迁移已评估。
4. 确认发布策略：灰度范围、观察窗口、回滚方案。
5. 发布后观察核心指标并记录结论。

## 检查清单

- [ ] CI 通过且无阻断告警
- [ ] 数据库变更有备份与回滚说明
- [ ] 关键接口健康检查正常
- [ ] 监控告警阈值已确认
- [ ] 发布记录已沉淀到 docs/sprints
