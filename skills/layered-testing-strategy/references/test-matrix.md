# 分层测试策略矩阵

## 执行步骤

1. 识别本次迭代的关键变更点。
2. 将每个变更映射到 unit、integration、e2e 三层。
3. 定义每层最小必测项与失败阈值。
4. 在 CI 中设置阻断策略。

## 建议矩阵

| 测试层 | 关注点 | 位置 | 阻断建议 |
| --- | --- | --- | --- |
| unit | 纯逻辑、边界值、工具函数 | `tests/unit`（规划中） | 必须阻断 |
| integration | 服务协作、数据库读写、外部依赖 | `tests/integration`（规划中） | 核心用例阻断 |
| e2e | 用户主链路、关键页面流程 | `tests/e2e`（规划中） | 主链路阻断 |

## 检查清单

- [ ] 新增业务规则有 unit
- [ ] 跨服务调用有 integration
- [ ] 关键用户路径有 e2e
- [ ] 失败用例可稳定复现
